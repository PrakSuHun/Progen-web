교수님의 모든 말씀을 녹음 + 텍스트화하고, 흩어진 자료를 NotebookLM으로 통합 가공한 뒤, 유니브AI로 문제 풀이·오답노트까지 만든 다음, 그 오답노트를 다시 NotebookLM 오디오 오버뷰로 변환해 시험장 가는 길에 듣는 것이 핵심.

1다글로 (Daglo) — 휴대폰 하나로 끝내는 녹음 AI

스마트폰만 있으면 시간표 등록부터 녹음·전사·요약·질의응답까지 한 앱으로 종결되는 모바일 특화 툴.

🌟 다글로가 좋은 이유

핵심 강점
  • 휴대폰 하나로 종결 — 별도 노트북·맥북 없이 폰만 들고 다니면 됨.
  • 수업 시작 1분 전 푸시 알림 — "지금 강의 녹음을 시작할까요?" 자동 알림.
  • 이미지로 시간표 자동 등록 — 에브리타임 시간표 스크린샷만 올리면 AI가 인식.
  • 녹음 → 스크립트 → 요약 → 보드챗이 한 앱 안에서 흐른다.
  • ChatGPT/Gemini/Claude 엔진 선택 기능 내장.

📱 활용 팁 1 — 시간표 이미지 등록

달력 아이콘 → [시간표 추가하기] → 이미지로 추가. 에브리타임 캡처를 그대로 올리면 끝.

📱 활용 팁 2 — 시간표가 곧 자동 알림

💡 핵심: 시간표 등록 = 수업 시작 1분 전 알림 자동 세팅. 출석 부를 때 푸시 알림 한 번만 터치하면 즉시 녹음 시작.

📱 활용 팁 3 — 보드챗으로 어려운 부분 즉시 풀이

일반 챗봇과 달리 내가 녹음한 수업 스크립트만 근거로 답변 → 환각 거의 없음.

예시: 방금 교수님이 설명하신 상대성 이론, 초등학생도 이해할 수 있게 비유로 설명해 줘.

📱 활용 팁 4 — 템플릿 요약 ('강의 노트' 형식)

요약 형식을 강의 노트 / 회의록 / 인터뷰 등에서 선택. 대학생은 강의 노트 템플릿 한 번이면 표·리스트로 정리된 복습 자료 완성.

📋 다글로 보드챗 프롬프트 모음 ⭐

📋 ① 수업 직후 5분 복습 노트 역할: 너는 방금 수업을 같이 들은 스터디 메이트야. 수업이 끝난 직후 5분 안에 머릿속에 핵심을 박아넣을 수 있도록 도와줘. 지시사항: 방금 녹음된 스크립트를 분석해서 아래 4가지를 정확히 이 순서대로 정리해줘. 1. 오늘 수업 한 줄 요약 (트위터 길이, 280자 이내) 2. 핵심 개념 3개 — 각 개념마다 정의 + 왜 중요한지 1문장 3. 교수님이 강조하거나 반복한 표현 / 시험 힌트 (있으면 원문 그대로 인용) 4. 내가 더 찾아봐야 할 모호한 부분 (스크립트만으론 이해가 부족한 지점) 제약: 스크립트에 없는 내용은 절대 추가 금지. 4번에서 모르는 건 솔직히 "확인 필요"라고 적어. 출력: 마크다운, 각 항목 굵은 소제목 사용.
📋 ② 시험 예상 문제 자동 추출 역할: 너는 출제 경향 분석 10년 차 조교야. 교수님 수업 스크립트만 보고도 어떤 문제가 시험에 나올지 정확하게 짚어내는 사람이야. 지시사항: 이 수업 스크립트에서 시험에 나올 가능성이 높은 포인트를 다음 기준으로 뽑아줘. - 교수님이 "중요하다 / 시험에 나온다 / 외워라 / 정리해두자" 같은 표현을 쓴 부분 - 같은 개념을 두 번 이상 반복한 부분 - 비교/대조/원인-결과 구조로 설명한 부분 - 정의를 천천히·정확하게 말한 부분 각 포인트마다 다음 형태로 정리: | 출제 예상 포인트 | 근거가 된 교수님 발언 (원문 인용) | 예상 출제 형태 (서술형/객관식/단답형) | 제약: 인용은 스크립트 원문 그대로. 추측·상상 금지. 근거가 약하면 빼.
📋 ③ 어려운 개념을 비유로 풀어달라 방금 교수님이 [개념명]에 대해 설명한 부분을 다시 읽어줘. 그리고 그 개념을 아래 두 단계로 풀어줘. 1단계 (초등학생용): 일상 비유 한 가지로 1~2문장 2단계 (대학생 시험용): 정의 + 핵심 메커니즘 + 자주 나오는 함정 마지막 줄에는 "이 개념이 시험에 나온다면 어떤 형태로 물을 가능성이 높은지" 한 줄로 예측. 제약: 비유는 강의 맥락과 어긋나지 않도록. 교수님이 사용한 용어/예시가 있으면 우선 활용.

⚠ 가성비 사용법 (무료 크레딧 2,000)

  • 녹음/전사 자체는 거의 무료 — 100시간 녹음해도 무료.
  • 요약·보드챗만 크레딧 차감.
  • 꿀팁: 크레딧 절약하려면 스크립트 복사 → ChatGPT/Gemini에 붙여넣어 무료로 요약.

2ALT — 카이스트가 만든 무료·무제한 끝판왕

카이스트 공대생들이 본인들 쓰려고 만든 완전 무료·무제한 녹음/학습 도구. 단, Mac/iPad 전용이라 컴퓨터 사양이 받쳐줘야 함.

🌟 ALT가 좋은 이유

핵심 강점
  • AI 요약·전사 완전 무료·무제한 — 다글로 크레딧 한계 없음.
  • 슬라이드 + 녹음 + 필기 3분할 화면 — 한 앱에서 학습 환경 완성.
  • 공대 수식 정리 퀄리티 압도적 — 카이스트 공대생 작품답게.
  • 다국어 실시간 번역 — 영어 강의·교환학생용으로 강력.
⚠ 단점: Mac / iPad 전용. Windows·Android 유저는 다글로로 대체.

💻 활용 팁 1 — 3분할 학습 환경

새 노트 생성 → 과목명 입력 → PDF 슬라이드 추가. 중앙=PDF, 우측=실시간 전사, 하단=메모장.

💻 활용 팁 2 — 사이버 강의 정복 (시스템 오디오 녹음) ⭐

녹음창 하단의 시스템 오디오 활성화 → 컴퓨터에서 나오는 모든 소리(싸강·Zoom)를 실시간 전사.

🎯 사이버 강의 활용 시나리오

  1. 싸강 영상 재생 + ALT 시스템 오디오 녹음 ON
  2. 그 시간에 다른 과제·축구 시청 등 자유롭게
  3. 수업 종료 후 ALT가 만들어둔 완벽한 스크립트 + AI 요약 1분 확인
  4. 영상 되돌리기 시간 0분, 데이터는 100% 확보

→ 남들보다 5배 빠른 데이터 수집의 시작점.

💻 활용 팁 3 — AI 요약 & 보드챗

우측 상단 [요약] 탭 → 핵심 키워드 + 수식 정리. 보드챗 질문 예시:

  • 오늘 배운 내용 중 가장 중요한 3가지를 질문 리스트로 만들어줘
  • 이 개념을 초등학생도 이해할 수 있게 다시 설명해줘
  • 방금 수업한 내용으로 퀴즈 10개 만들어줘
  • (틀린 후) 이 문제 왜 틀렸어? 교수님은 어떻게 설명하셨어?

📋 ALT 보드챗 프롬프트 모음 ⭐ (공대·이과 특화)

📋 ① 수식·증명 정리 (공대 강의 전용) 역할: 너는 공대 대학원생 조교야. 교수님이 칠판에 풀어낸 수식과 증명을, 학생이 시험장에서 막힘없이 재현할 수 있도록 정리해주는 사람이야. 지시사항: 이번 수업 스크립트와 슬라이드(PDF)에서 등장한 수식·정리·증명을 모두 추출해서 아래 구조로 정리해줘. 각 수식마다: 1. 수식 이름 / 정리 이름 (없으면 적절한 이름 부여) 2. 수식 자체 (LaTeX 또는 일반 표기) 3. 각 기호가 무엇을 뜻하는지 한 줄씩 4. 이 수식이 어떤 상황/조건에서 성립하는지 5. 교수님이 강조한 유도 과정의 핵심 한 줄 6. 시험에 나온다면 어떤 형태일지 (계산형 / 증명형 / 적용형) 제약: 스크립트와 슬라이드에 등장한 수식만 다뤄. 외부 일반 지식으로 채워 넣기 금지. 출력: 마크다운, 수식별 카드 형태.
📋 ② 사이버 강의 30분 압축 요약 역할: 너는 시간 없는 시험 직전 학생을 위한 압축 요약 전문가야. 1~2시간짜리 사이버 강의를 듣지 못한 친구가 30분 안에 핵심을 따라잡을 수 있도록 정리해줘. 출력 구조: 1. ⏱ 한 줄 요약 (이 강의의 결론) 2. 🗺 이 강의 흐름도 (3~5단계, 화살표로 연결) 3. 📌 핵심 개념 5개 — 정의 + 한 줄 설명 4. ⚠ 교수님이 두 번 이상 강조한 부분 (시험 신호) 5. ❓ 보기 전 vs 본 후의 차이 — 이 강의를 안 봤다면 놓쳤을 핵심 한 가지 6. 🎯 다음 강의에 이어질 것으로 보이는 떡밥 제약: 스크립트 외부 지식 사용 금지. 시간 표기는 hh:mm:ss 형태로 인용.
📋 ③ 즉석 퀴즈 + 채점 방금 강의 내용으로 다음 형식의 퀴즈 10개를 만들고, 정답·해설·근거 위치를 함께 줘. 형식: [객관식 5 / OX 3 / 단답형 2] 각 문항: - 문제 - 보기 (객관식만) - 정답 - 해설 (3~4문장) - 근거: 스크립트 mm:ss 또는 슬라이드 페이지 내가 답을 보내면 채점하고, 틀린 문항은: 1. 왜 틀렸는지 진단 (오개념/계산 실수/정의 오해 중 분류) 2. 같은 유형 추가 문제 1개 더 출제 이 흐름으로 진행해줘.

3NotebookLM — 흩어진 자료를 하나로 가공

다글로/ALT로 모은 녹음 스크립트 + 교수님 PDF 교안을 NotebookLM에 통째로 넣어 나만의 1:1 AI 연구원으로 만든다.

🌟 NotebookLM이 좋은 이유

핵심 강점
  • 업로드한 소스 안에서만 답 — 환각 없음, 신뢰도 100%.
  • 교수님이 원하는 풀이로 정확히 공부 가능 — 일반 GPT는 인터넷 일반 지식으로 답해서 교수님 논리와 어긋남.
  • 교수님이 강조한 그 문장·그 키워드 그대로 학습 → 출제자의 머릿속을 들여다보는 효과.
  • 여러 PDF·스크립트를 통합 분석한 요약 생성.

📚 가공 STEP 1 — 소스 통합 업로드

  • 새 노트북 생성 → 교수님 PDF 교안 + 다글로/ALT로 뽑은 스크립트를 모두 업로드.
  • 스크립트는 PDF 변환 X — 텍스트 복사 붙여넣기로 바로 소스 등록 가능.

📚 가공 STEP 2 — 시험 대비 요약본 생성 (프롬프트 ⭐)

중앙 하단 프롬프트 입력창에 아래 프롬프트를 그대로 복사해 던지면 교재 다시 펼칠 필요 없는 수준의 요약본이 생성된다.

📋 NotebookLM 요약본 생성 프롬프트 역할: 너는 시험 2주 전 대학생을 위한 '시험 대비 핵심 요약 튜터'야. 10년 경력의 학습 코치처럼, 이 요약본만 읽으면 교재를 다시 펼칠 필요 없을 정도로 자세하고 체계적으로 정리해주는 전문가야. 배경: 내가 업로드한 문서는 이번 시험 범위의 강의자료/교재야. 시험이 2주 남았고, 이 요약본 하나로 전체 내용을 공부할 수 있어야 해. 지시사항: 문서 전체를 분석해서 아래 구조로 한 번에 정리해줘. 1. 전체 흐름 요약: 문서가 다루는 내용의 큰 흐름을 3~5문장으로 먼저 잡아줘. 숲을 먼저 보여주는 거야. 2. 단원/주제별 상세 정리: 문서의 구조를 따라가면서 각 단원(또는 주제)마다 아래 형태로 정리해줘. - 핵심 개념 설명: 각 개념이 뭔지, 왜 나왔는지, 어떻게 작동하는지를 교과서 수준으로 자세히 서술. 단순 키워드 나열 금지. - 예시/사례: 문서에 예시가 있으면 반드시 포함. - 중요 수식/모델/이론: 있으면 정리하고, 각 요소가 뜻하는 바를 풀어서 설명. 3. 개념 간 관계 & 비교표: 서로 연결되거나 헷갈리기 쉬운 개념들을 | 구분 | A | B | 형태의 표로 비교 정리. 4. 시험 출제 예상 포인트: 비교/대조, 정의, 원인-결과, 장단점 등 시험에 나올 만한 내용을 bullet으로 짚어줘. 제약 조건: - 반드시 업로드된 문서 내용만 사용해. 외부 지식 절대 섞지 마. - 전문 용어는 괄호 안에 쉬운 말로 풀어줘. - 내용을 축약하느라 의미가 빠지면 안 돼. 길어도 괜찮으니 공부할 수 있을 만큼 충분히 써줘. - 문서에서 확인 안 되는 건 "문서에서 확인할 수 없음"이라고 써. 출력 형식: 마크다운. 표, 소제목, bullet을 활용해서 시각적으로 깔끔하게. 이 결과물을 프린트해서 바로 공부할 수 있는 수준으로.

📚 가공 STEP 3 — 동영상 개요(나만의 인강)

📋 NotebookLM 추가 프롬프트 모음 ⭐

📋 ① 헷갈리는 개념 비교표 자동 생성 역할: 너는 시험에서 학생들이 가장 자주 혼동하는 개념을 알아채는 베테랑 강사야. 지시사항: 업로드된 자료 전체를 훑어서, 학생이 헷갈리기 쉬운 개념 쌍(또는 그룹)을 5~8쌍 뽑아 비교표로 정리해줘. 각 비교 항목마다: | 구분 | 개념 A | 개념 B | (있다면 개념 C) | |---|---|---|---| | 정의 | | | | | 핵심 차이 | | | | | 자주 혼동되는 이유 | | | | | 시험에서 묻는 방식 | | | | | 근거 (소스의 어느 페이지/시점) | | | | 마지막에 "이 표만 외워도 객관식 X점은 확보된다"는 평을 한 줄 덧붙여줘. 제약: 업로드 소스에 명시적으로 등장한 개념만. 외부 일반 지식으로 비교 항목 채우기 금지.
📋 ② 족보·기출 분석 (지난 시험지가 있을 때) 역할: 너는 출제 패턴을 분석하는 시험 분석가야. 배경: 업로드된 자료 중 [기출문제 / 과제 / 퀴즈] 파일을 [강의자료]와 대조 분석해줘. 분석 결과 출력: 1. 기출 문항별 → 강의자료의 어느 부분(단원·페이지·시점)에서 출제됐는지 매핑 2. 출제 빈도가 높은 개념 TOP 5 (반복 등장 순) 3. 출제되지 않았지만 강의에서 강하게 강조된 — "이번 시험 신규 출제 가능성 높은" 개념 TOP 5 4. 문제 유형 비율 (개념 정의 / 비교 대조 / 계산 / 사례 적용 / 서술) 5. 권장 공부 우선순위 (시간 대비 효율 기준) 제약: 업로드된 자료 외부의 기출이나 일반 지식은 사용 금지.
📋 ③ 마인드맵 텍스트 생성 업로드된 자료 전체를 바탕으로, 시험 범위 전체의 마인드맵을 텍스트로 그려줘. 형식: # 중심 주제 ## 1. 대주제 - 1.1 개념 — 한 줄 정의 - 1.1.1 세부 — 핵심 키워드 - 1.2 개념 — 한 줄 정의 ## 2. 대주제 ... 각 노드 옆에 [근거: 페이지/시점] 표기. 마지막에 "이 마인드맵에서 가장 약한 가지 = 자료에서 설명이 부족했던 영역" 1~2개를 짚어줘.

4유니브 AI — 한국 대학생 특화 실전 정복

밤샘 시험공부를 1시간으로 끝내는 한국 대학 시험 특화 끝판왕.

🌟 유니브AI가 좋은 이유

핵심 강점
  • 한국 대학 시험 유형 최적화 — 객관식·OX·단답·빈칸까지.
  • 한 번에 30문제 자동 출제 — 기출 족보 한 세트 분량.
  • 틀린 문제 → 원문 PDF 페이지 자동 점프.
  • 오답노트 PDF 자동 추출 — 시험 직전 5분용 단권화 노트.
  • 마인드맵·플래시카드·AI 튜터(올가미 캡쳐)까지 풀세트.

🎯 사용법 (간단 정리)

  1. 폴더 생성 → 시험 과목명 입력
  2. 해당 과목 PDF 한 번에 하나씩 업로드
  3. 좌=원문 / 우=요약 / [쉬운 설명] · [AI 튜터 드래그] 활용
  4. 특정 페이지만 강조됐다면 +추가 요약 생성 → 페이지 지정(예: 10~13)
  5. 퀴즈 생성 → 풀이 → 틀린 문제 복습 → PDF 추출로 오답노트 저장

🎯 핵심 기능 1 — AI 슬라이드 / 화면 캡쳐

그래프·도표·수식처럼 텍스트 드래그가 안 되는 부분은 올가미 캡쳐 → AI에게 시각 분석 요청.

🎯 핵심 기능 2 — 오답노트 PDF 추출 ⭐

복습하기 → 틀린 문제만 풀이 → PDF 추출 버튼 → 1초 만에 나만의 오답노트 PDF 생성.

🎯 초특급 꿀팁 — 유니브 ✕ NotebookLM 합체

유니브에서 뽑은 오답노트 PDF를 NotebookLM에 다시 업로드 → 오디오 오버뷰로 변환.
시험장 가는 길 / 버스 / 지하철에서 가장 헷갈렸던 오답들만 귀로 학습.

🎯 운용 순서 한 줄 요약

눈으로는 유니브 오답노트 PDF, 귀로는 NotebookLM 오디오 오버뷰 → 시험장 입성.

📚 NotebookLM 오디오 오버뷰 프롬프트 (오답노트 전용) ⭐

NotebookLM에서 오답노트 PDF 업로드 후 스튜디오 → 오디오 개요 → 맞춤 설정에 아래 프롬프트를 그대로 입력.

🎙 NotebookLM 오답노트 오버뷰 프롬프트 이 자료는 대학생의 시험 오답노트입니다. 두 명의 진행자가 친근한 대화 형식으로, 각 오답 문제를 하나씩 짚어가며 다음 흐름으로 설명해 주세요: 1. 문제가 무엇을 물어보고 있었는지 핵심만 짧게 요약 2. 학생이 왜 틀렸을 가능성이 높은지 (흔한 오개념, 함정, 헷갈리는 포인트) 분석 3. 올바른 풀이 과정과 정답을 명확히 설명 4. 비슷한 유형이 나왔을 때 다시 안 틀리려면 기억해야 할 핵심 포인트 한 줄 정리 톤은 같이 공부하는 선배가 알려주는 느낌으로, 쉽고 자연스럽게 해 주세요. 어려운 개념은 일상적인 비유를 활용해서 설명해 주세요.

📋 유니브 / NotebookLM 오디오 추가 프롬프트 모음 ⭐

📋 ① 유니브 AI 튜터 — 약점 진단 방금 푼 30문제 결과를 바탕으로, 내가 어느 개념에서 약한지 진단해줘. 출력 구조: 1. 정답률이 낮은 개념 TOP 3 — 각 개념의 정의를 한 번 더 풀이 2. 같은 개념 안에서 어떤 유형(정의/적용/비교/계산)에 약한지 3. 권장 복습 동선: "①번 개념 → 원문 X페이지 다시 읽기 → 추가 5문제 풀기" 순서로 4. 만약 시험까지 1시간만 남았다면 어디부터 봐야 하는지 우선순위 한 줄 제약: 채점 결과 데이터 안에서만 판단. 일반론 금지.
📋 ② NotebookLM 오디오 — '큰 그림' 버전 (오답노트 외) 이 자료는 시험 범위 전체 강의자료입니다. 두 진행자가 친근한 대화체로, 학생이 시험 범위 전체를 한 번에 머릿속에 그릴 수 있도록 다음 흐름으로 설명해 주세요. 1. 인트로: 이 과목이 결국 무엇을 다루는 학문인지 한 줄로 잡기 2. 큰 줄기: 시험 범위를 3~4개의 큰 단원으로 묶고, 각 단원이 왜 존재하는지(왜 배우는지) 설명 3. 단원 간 연결: A 단원이 B 단원을 어떻게 받쳐주는지 — 흐름과 인과관계 4. 함정 포인트: 학생들이 자주 헷갈리는 부분 2~3가지 5. 마지막 한 마디: "시험장 들어가기 전에 이것만 기억해라" 한 문장 톤은 같이 시험 보는 동기/선배 느낌. 비유는 일상적인 것 사용. 길이는 12~15분 정도가 이상적.
📋 ③ NotebookLM 오디오 — '플래시카드' 버전 이 자료(또는 오답노트)에서 핵심 용어 30개를 뽑아, 두 진행자가 다음 형식으로 진행하는 플래시카드형 오디오를 만들어주세요. 각 용어마다: A 진행자: "다음 용어 — [용어]" A 진행자: "(2초 침묵)" — 청취자가 답을 떠올릴 시간 B 진행자: "정의는 [정의]. 핵심은 [한 줄 핵심]. 자주 헷갈리는 건 [혼동 포인트]." A 진행자: "다음." 마지막에는 30개 중 가장 시험에 잘 나올 것 같은 5개를 한 번 더 빠르게 복습. 톤: 단조롭지 않게, 적당한 텐션. 30분 이내.
