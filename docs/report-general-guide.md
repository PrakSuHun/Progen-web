# 일반 보고서 작성 가이드

## 용도
**대외적으로 사용할 보고서**입니다.
외부 파트너, 학교, 후원자, 언론 등에게 공유할 수 있는 순수 커뮤니티 운영 보고서입니다.
PROGEN은 대외적으로 "AI·자동화 대학생 커뮤니티"이며, 포도/선교 맥락은 외부에 드러나면 안 됩니다.

## 기본 원칙
종교색 없는 순수 커뮤니티 운영 보고서로 작성합니다.

## 금지 단어
- **포도** (정회원) — 절대 사용 금지
- **정회원** — 사용 금지
- **선교**, **전도**, **기도** 등 종교 관련 용어 — 사용 금지

## 데이터 처리 규칙 (매우 중요!)

### 핵심 원칙
**모든 수치는 포도(정회원)를 포함한 총 크루 기준**으로 계산합니다.
포도는 외부에 존재하지 않는 개념이므로, 포도도 일반 크루원과 동일하게 취급합니다.

### 구체적 예시

**잘못된 예 (포도 제외) ❌**
- 크루 신청 17명 (포도 4명 미포함)
- 크루 출석 11명 (포도 4명 미포함)
- 누적 크루 56명 (포도 미포함)

**올바른 예 (포도 포함) ✅**
- 크루 신청 **21명** (17 + 4)
- 크루 출석 **15명** (11 + 4)
- 누적 크루 **약 79명** (비포도 57 + 포도 22)
- 전체 신청 **53명** (일반 49 + 포도 4)
- 전체 출석 **32명** (일반 28 + 포도 4)

### 집계 기준
- **전체 신청 = 모든 event_registrations** (포도 포함)
- **전체 출석 = 모든 status='출석완료'** (포도 포함)
- **크루 신청 = crew_id로 등록된 전원** (is_member 필터 없음)
- **크루 출석 = crew_id + 출석완료** (is_member 필터 없음)
- **누적 크루 = crew_members 전체 수** (is_member 필터 없음)
- **게스트 = guest_id로 등록된 전원** (그대로)

### 변경 표기
- 1강 대비 변화 표기 시에도 포도 포함 수치끼리 비교
- 분석 탭의 숫자(포도 제외)를 그대로 쓰면 안 됨 — 반드시 포도 포함하여 재계산

## 주의사항
- 분석 탭/대시보드의 수치는 **포도 제외** 수치임 (내부 운영용)
- 보고서 작성 시 **DB에서 직접 포도 포함 수치를 재집계**하거나, 분석 탭 수치 + 포도 수를 더해서 사용
- crew_members 테이블에서 `is_member` 필터를 사용하지 말고 전체를 "크루"로 집계
- event_registrations에서도 crew_id가 있는 전원을 "크루"로 집계 (is_member 무관)

## 보고서 구조 (표준)

1. **행사 개요** — 날짜, 주관, 대상, 형태, 핵심 수치 요약
2. **참가자 현황 분석** — 대학별, 학년별, 성별 분포
3. **신청 경로 분석** — 경로별 신청/출석률
4. **참가자 피드백 분석** — 강점, 개선점, 재참여 의향
5. **크루 회원화 분석** — 신청→가입 전환 퍼널, 누적 크루 현황
6. **이전 행사 대비 변화 분석** — 주요 지표 비교
7. **핵심 인사이트 및 향후 제언** — 인사이트 5개, 실행 제언
8. **종합 요약** — 핵심 강점 / 핵심 약점
